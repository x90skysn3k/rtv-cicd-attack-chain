What you need

  • A GitHub account.
  • A terminal with aws, curl, and jq.
  • Internet access to github.com, api.github.com, and *.amazonaws.com.
  • No AWS account of your own for the live training flow.

Core commands

After your workflow log prints STS credentials, paste the four export lines into your terminal. Then run:

PAT=$(aws secretsmanager get-secret-value \
  --secret-id demo/github-pat \
  --query SecretString --output text)

curl -X PUT \
  -H "Authorization: token $PAT" \
  -H "Accept: application/vnd.github+json" \
  "https://api.github.com/repos/pipeline-demo-lab/cicd-demo/pulls/<PR_NUMBER>/merge"

Safety notes

  • The live demo uses a dedicated AWS account and throwaway GitHub organization.
  • The attendee role can read exactly one Secrets Manager secret.
  • STS sessions expire after 15 minutes.
  • Do not reuse this pattern in any real repository.
